# Work 工作目录系统

> **"目录是静态导航，搜索是动态索引。"** —— 亦仁益语
>  **"你越懒，系统越要聪明。"** 让目录自己"长"出来，而不是靠记忆去找。

## 📁 目录结构

```
/work
│
├─ 00_Inbox/              # 数字收件箱：任意临时放入，每周清空
│
├─ 01_Brain/              # 个人知识库：笔记、卡片、阅读摘要
│   └─ 00_Temp/          # 临时思考，未分类整理
│
├─ 02_Projects/           # 项目管理中心
│   ├─ _scaffold/        # 项目模板和初始化脚本
│   ├─ 01_Active/        # 进行中项目
│   ├─ 02_Planning/      # 计划中项目
│   ├─ 03_Paused/        # 暂停项目
│   └─ 04_Done/          # 完成项目（待归档）
│
├─ 03_Assets/            # 通用素材：图库、ICON、PPT模板等
│
├─ 04_Tools/             # 可执行脚本、工具集合
│
├─ 90_Reference/         # 规范、合同等 rarely-change 文件
│
├─ 98_Sandbox/           # 快速试验场：随时清理
│
└─ 99_Archive/           # 归档区：按年季度分类
    └─ 2025/
        ├─ Q3/
        └─ Q4/
```

## 🎯 设计原则

### 核心原则

| 原则         | 说明                                     | 典型做法                                         |
| ------------ | ---------------------------------------- | ------------------------------------------------ |
| **单一职责** | 每个文件夹只做一件事，查找时心智负担最小 | `/code` 只放源码、`/doc` 只放文档                |
| **层级不深** | 理想 3 层封顶，少点"套娃"                | `/work/02_Projects/01_Active/proj-name` → 深度 3 |
| **可检索化** | 文件/文件夹名即关键词，配合全局搜索      | 日期 + 语义：`2025-07-21_日报.md`                |
| **临时隔离** | 杂物都有"回收站"，别塞到正式目录         | `00_Inbox`、`98_Sandbox` 随时清理                |
| **自动化**   | 用脚本/模板生成目录，保证一致性          | `init_project.sh` 一键创建项目                   |

### 工作流对应

```
收集 → 加工 → 产出 → 归档
 ↓      ↓      ↓      ↓
00_   01_    02_    99_
Inbox Brain Projects Archive
```

## 📝 命名规则

| 场景   | 推荐格式                | 例子                     |
| ------ | ----------------------- | ------------------------ |
| 文件   | `YYYY-MM-DD_关键词.ext` | `2025-07-21_日报.md`     |
| 周期物 | `YYYY-WW_周报.md`       | `2025-W30_周报.md`       |
| 版本   | `vMAJOR.MINOR.PATCH`    | `v1.4.2_设计稿.sketch`   |
| 项目   | `proj-业务-关键词`      | `proj-ai-isolaze-beacon` |

**要点：**

- 全英文 + 下划线/中划线，避免空格和中文路径
- 命名先时间再语义，排序与时间线一致
- 数字前缀 = 排序 + 阶段提示

## 🔧 项目管理流程

### 项目状态流转

```
Planning → Active → Done → Archive
   ↑         ↓        ↓       ↓
 新想法    执行中   待归档   季度存档
   ↓         ↓
 Paused   临时暂停
```

### 项目创建

使用 `_scaffold/init_project.sh` 快速创建标准项目结构：

```bash
cd /work/02_Projects/_scaffold
./init_project.sh proj-example "项目描述"
```

自动生成：

- 标准目录结构（code/doc/design/output 等）
- README.md 模板
- 项目配置文件

## 📅 维护节奏

| 频率         | 动作                                     | 工具/说明       |
| ------------ | ---------------------------------------- | --------------- |
| **每日收工** | 清空 `00_Inbox`，移动到对应文件夹        | Alfred / fzf    |
| **每周五**   | `git commit` & 清理 `98_Sandbox`         | 备份 & 清理杂物 |
| **每月初**   | 将完结项目从 `04_Done` 移至 `99_Archive` | 手动或脚本      |
| **每季度**   | 审计 `03_Assets` 是否重复或过期          | `du -sh`        |

## 🛠️ 推荐工具链

### 必备工具

- **Git**: 管理整个 `/work` 目录（大文件用 Git LFS）
- **Alfred/fzf**: 快速文件搜索和导航
- **VS Code**: 通用编辑器，支持全目录搜索
- **Obsidian/Logseq**: 知识管理（用于标签，不依赖目录结构）

### 自动化脚本

放置在 `04_Tools/` 目录：

- 项目初始化脚本
- 定期清理脚本
- 归档整理脚本
- 备份同步脚本

## 📋 最佳实践

### README 优先

每个重要目录都应有 README.md，说明：

- 这里存放什么
- 为什么需要这个目录
- 如何使用和维护

### 标签而非分类

对于多维度信息：

- 用 Obsidian/Logseq 做标签管理
- 避免在文件系统里硬拗多级目录
- 文件系统专注结构化，标签系统专注语义化

### 版本控制

```bash
# 推荐工作流
git add .
git commit -m "Weekly snapshot $(date +%Y-%m-%d)"
git clean -fd 98_Sandbox/  # 清理临时文件
```

## 🚀 快速开始

1. **新建项目**

   ```bash
   cd 02_Projects/_scaffold
   ./init_project.sh proj-new-idea "我的新想法"
   ```

2. **日常收集**

   - 临时内容 → `00_Inbox/`
   - 想法笔记 → `01_Brain/00_Temp/`
   - 试验代码 → `98_Sandbox/`

3. **定期整理**

   - 每周处理 Inbox
   - 每月整理归档
   - 每季度大扫除

## 💡 进阶技巧

### 全局搜索优化

利用文件名关键词：

```bash
# 快速找到特定日期的文件
find . -name "*2025-07-21*"

# 按类型搜索
find . -name "*日报*"
find . -name "*会议*"
```

### 自定义别名

在 `.bashrc` 或 `.zshrc` 中添加：

```bash
alias cdw="cd ~/work"
alias cdi="cd ~/work/00_Inbox" 
alias cdp="cd ~/work/02_Projects/01_Active"
```
